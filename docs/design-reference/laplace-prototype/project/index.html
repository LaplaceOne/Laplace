Intent-based atomic settlement · Solana · Devnet

Escrow that releases only on proof, refunds on expiry.

Laplace is a Solana protocol for intent-based atomic settlement. A maker locks assets in an escrow that releases to a recipient only when a pluggable on-chain criterion is provably satisfied before expiry — otherwise the maker is refunded.

Guarantees

Enforced by the program,
not by trust.

Every property below holds on-chain — there is no operator, relayer, or admin key in a position to weaken them.

Non-custodial

Assets sit in a program-derived escrow from the moment they're locked. No operator, relayer, or admin key can move them.

Atomic

Settlement is a single instruction — release to the receiver or refund the maker. Never a partial transfer, never a stuck state.

Refund-guaranteed

If no valid fulfillment lands by the expiry slot, the refund recipient reclaims the escrow in full. A failed intent is never a lost deposit.

How it works

The lifecycle
is the product.

Every Laplace intent is a small on-chain state machine: create → fulfill | refund → close. The escrow core is criterion-agnostic. Read the protocol

01

Create

The maker locks SOL or SPL into an intent PDA and binds a criterion program, receiver, refund recipient, and expiry slot. Funds are non-custodial from this point on.

02

Fulfill

A fulfiller submits evidence. The core forwards a canonical request to the criterion program by CPI and releases the escrow to the receiver only if that CPI accepts — atomically.

03

Refund or close

No valid fulfillment by expiry? Anyone can crank the refund to the refund recipient. After a terminal state, the maker closes the intent to reclaim rent.

Pluggable criteria

One protocol,
many faces.

A criterion is any program implementing the verify_criterion interface. Two ship today; more are named in the design. All criteria

Hashlock

official

Release when a preimage hashing to a committed digest is revealed on-chain. Stateless — the primitive behind trustless atomic swaps and public unlocks.

SHA256(secret) == criterion_data_hash

Validity · SP1

official

Release against a Groth16 proof verified on-chain by a committed verifying key. Settle on private or complex computation without revealing the witness.

sp1_verify(proof, vkey_hash)

Named for future releases — each a criterion program plus a front-end, never a fork of the core: Signature, Encrypted disclosure, Cross-chain lock proof, Multi-criterion.

Roadmap

Laplace Bridge

Cross-chain atomic swap. Two chains running Laplace swap tokens with a shared hashlock and asymmetric timeouts; relayers add discovery and propagation, never custody.

Roadmap

Laplace Disclosure

Buy a verifiable secret. A seller is paid only after publishing an encrypted secret plus an SP1 proof that the plaintext satisfies the buyer's criteria and is bound to that ciphertext.

For developers

A single typed SDK
talks to the chain.

UIs never hand-build instructions. The SDK derives PDAs, assembles criterion + settlement accounts, and pins the protocol constants. Role discovery is client-side, swappable for an indexer.

SDK quickstart GitHub
create-intent.ts
// lock 1,200 USDC, release on a revealed secret
const intent = await laplace.createIntent({
  asset:     { spl: USDC }, amount: 1_200e6,
  receiver:  bob, refundRecipient: alice,
  criterion: Condition.hashlock({ secret }),
  expirySlot: currentSlot + minutesToSlots(60),
});

// anyone reveals the preimage → atomic release
await laplace.fulfillIntent(intent, { secret });
